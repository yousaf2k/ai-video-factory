# Why ThenVsNow Has No Shots Agent or Sub-LLM Calls

**Date:** March 12, 2026
**Status:** By Design

---

## Overview

The ThenVsNow workflow is **intentionally streamlined** compared to the standard documentary workflow. It uses **only ONE LLM call** (the story agent) instead of multiple agents.

---

## Standard Documentary Workflow (Multiple Agents)

```
User Idea
    ↓
┌─────────────────────────────────────┐
│ Story Agent (Pass 1: Master Script)    │  ← LLM Call #1
│ "Write a master script for this idea"│
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ Story Agent (Pass 2: Scene Breakdown) │  ← LLM Call #2
│ "Break master script into scenes"    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ Session Service (Save story.json)    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ Shots Agent (Plan Individual Shots)  │  ← LLM Call #3
│ "Plan shots for each scene"         │
└─────────────────────────────────────┘
    ↓
Result: story.json + shots.json
```

**Total LLM Calls:** 3 (Story → Story → Shots)

---

## ThenVsNow Workflow (Single Agent)

```
User Movie Name
    ↓
┌─────────────────────────────────────┐
│ ThenVsNow Story Agent               │  ← LLM Call #1 (ONLY CALL)
│ "Generate complete reunion story   │
│  with characters, scenes, shots,    │
│  prompts, and metadata"             │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ generate_shots_from_then_vs_now()   │  ← Code Function (NO LLM)
│ "Transform story data into shots"    │
└─────────────────────────────────────┘
    ↓
Result: story.json + shots.json (all in one go)
```

**Total LLM Calls:** 1 (Story Agent Only)

---

## Why This Design?

### Reason 1: Structure is Fixed

**ThenVsNow has a predictable structure:**
- Movie name → Fixed set of iconic characters
- Each character has: THEN prompt, NOW prompt, Meeting prompt, Departure prompt
- Each character appears in 1-2 scenes
- Video format is standardized: Meeting + Departure videos per character

**No need for shots agent** because the structure is:
- Predictable (1 shot per character)
- Predefined (always Meeting + Departure videos)
- Template-based (same information for each character)

### Reason 2: All Prompts Generated at Once

The ThenVsNow story agent is designed to generate **EVERYTHING** in a single response:

```json
{
  "title": "Movie Name: The Reunion",
  "characters": [
    {
      "name": "Actor as Character",
      "then_prompt": "Young actor in original attire...",
      "now_prompt": "Actor today in expensive attire...",
      "meeting_prompt": "THEN MEETS NOW: Younger character...",
      "departure_prompt": "TRANSITION: Both characters walk..."
    }
  ],
  "scenes": [
    {
      "location": "Movie Set Location",
      "action": "Character walks onto set...",
      "set_prompt": "Detailed movie set background..."
    }
  ]
}
```

**No sub-LLM calls needed** because:
- Character prompts → Generated by story agent
- Scene prompts → Generated by story agent
- Video prompts → Generated by story agent
- Shot structure → Created by code, not LLM

### Reason 3: Cost & Speed

**Standard Documentary:**
```
3 LLM calls × $0.01-0.10 per call = $0.03-0.30
+ 3 API calls + 3 JSON parse operations
Total time: 30-60 seconds
```

**ThenVsNow:**
```
1 LLM call × $0.01-0.10 = $0.01-0.10
+ 1 API call + 1 JSON parse operation
Total time: 10-20 seconds
```

**Result:** 3x faster, 3x cheaper

## What About "Batches"?

You mentioned seeing "batches" - let me explain what's happening:

### In the Standard Workflow

**Shots Agent creates batches** for efficient generation:
```
Shot 1-5:  Batch 1 (First scene)
Shot 6-10: Batch 2 (Second scene)
Shot 11-15: Batch 3 (Third scene)
```

**Why:** To organize related shots and optimize parallel processing

### In the ThenVsNow Workflow

**No "batches" are created** because:
1. Shots are organized by character, not scenes
2. Each shot is independent (all use same FLFI2V workflow)
3. All shots can be generated in parallel if needed

**What you might be seeing:**

The web UI might display "batch" terminology, but this is just **UI grouping** for display purposes, not actual batch processing.

**Example:**
```
Web UI Display:
- Batch 1 (implied): Shots 1-5
- Batch 2 (implied): Shots 6-10

Actual Backend:
- All shots are processed individually
- No actual batch groups in code
```

## Code Evidence

### Standard Documentary (`core/story_engine.py`)

```python
def build_story(idea, agent_name="default", target_length=None):
    # Pass 1: Master Script (LLM Call #1)
    master_json_str = provider.ask(master_prompt, response_format="application/json")

    # Pass 2: Scene Breakdown (LLM Call #2)
    story_json_str = provider.ask(prompt, response_format="application/json")

    # Later...
    # Shots Agent (LLM Call #3)
    shots = plan_shots_from_scenes(story)  # Calls LLM
```

### ThenVsNow (`core/story_engine.py`)

```python
def build_story_then_vs_now(movie_name, target_length=None):
    # Single LLM Call (Story Agent)
    story_json_str = provider.ask(prompt, response_format="application/json")
    story = json.loads(story_json_str)

    # NO LLM - Code transforms story to shots
    shots = generate_shots_from_then_vs_now_story(story)
    story['shots'] = shots

    return story_json_str
```

## Key Takeaways

### ✅ This Is Intentional, Not a Bug

1. **ThenVsNow uses 1 agent, not 3** - by design
2. **No shots agent** - structure is fixed, not dynamic
3. **No sub-LLM calls** - everything generated in one shot
4. **Faster and cheaper** - streamlined for predictable format
5. **"Batches" you see** - likely UI grouping, not actual backend batches

### 🎯 When to Use Each Workflow

**Use Standard Documentary when:**
- Creating original content from ideas
- Need creative shot planning
- Want dynamic scene breakdowns
- Require custom shot compositions

**Use ThenVsNow when:**
- Making reunion videos for existing movies
- Structure is predictable (Character × 2 videos)
- Need fast, cost-effective generation
- Format is fixed (Meeting + Departure videos)

## Future Enhancement Possibilities

If you WANT sub-LLM calls in ThenVsNow, you could:

1. **Add Scene Agent** to generate detailed set backgrounds
2. **Add Shots Agent** to customize camera movements
3. **Add Character Agent** to develop character backstories

**But:** This would:
- Increase cost (3x more LLM calls)
- Increase generation time (3x longer)
- Add complexity without much benefit for the fixed format

**Current design is optimal for the use case.**

---

## Summary

The ThenVsNow workflow **intentionally skips** the shots agent and sub-LLM calls because:

1. ✅ **Structure is fixed** - 1 shot per character with 2 videos
2. ✅ **All prompts generated at once** - story agent creates everything
3. ✅ **Code creates shots** - no LLM needed for structured transformation
4. ✅ **3x faster** - single LLM call vs three
5. ✅ **3x cheaper** - reduces API costs significantly

**This is a feature, not a bug!** The streamlined workflow is optimized for the predictable reunion video format.
